Introduction Pendahuluan

The Crypto Payment Gateway API enables seamless integration of cryptocurrency payments into your applications. This documentation provides comprehensive information about available endpoints, authentication methods, and implementation examples.

API Gateway Pembayaran Crypto memungkinkan integrasi pembayaran cryptocurrency yang mulus ke dalam aplikasi Anda. Dokumentasi ini menyediakan informasi komprehensif tentang endpoint yang tersedia, metode autentikasi, dan contoh implementasi.

Authentication Autentikasi

All API requests require authentication using an API key. Include your API key in the request headers:

Semua permintaan API memerlukan autentikasi menggunakan kunci API. Sertakan kunci API Anda dalam header permintaan: 

Authorization: Bearer YOUR_API_KEY

API Endpoints Endpoint API

Create Payment Buat Pembayaran

POST

Create a new cryptocurrency payment request Buat permintaan pembayaran cryptocurrency baru 

POST /api/v1/payments

Request Body Body Permintaan 

{
    "amount": 100,
    "currency": "USD",
    "crypto": "BTC",
    "callback_url": "https://your-domain.com/callback"
}

Get Payment Status Cek Status Pembayaran

GET

Retrieve the status of a payment Mengambil status pembayaran 

GET /api/v1/payments/:id

Response Respons 

{
    "id": "payment_123",
    "status": "pending",
    "amount": "100",
    "currency": "USD",
    "crypto_amount": "0.0034",
    "crypto": "BTC",
    "address": "bc1q..."
}

Code Examples Contoh Kode

Create Payment Request Membuat Permintaan Pembayaran 

const response = await fetch('https://api.example.com/v1/payments', {
    method: 'POST',
    headers: {
        'Authorization': 'Bearer YOUR_API_KEY',
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        amount: 100,
        currency: 'USD',
        crypto: 'BTC',
        callback_url: 'https://your-domain.com/callback'
    })
});

Check Payment Status Memeriksa Status Pembayaran 

const response = await fetch('https://api.example.com/v1/payments/payment_123', {
    headers: {
        'Authorization': 'Bearer YOUR_API_KEY'
    }
});
